## <a name="motivations-behind-the-format"></a><span data-ttu-id="f7e51-104">Motivazioni alla base del formato</span><span class="sxs-lookup"><span data-stu-id="f7e51-104">Motivations behind the format</span></span>

<span data-ttu-id="f7e51-105">All'inizio del 2016 diversi team Microsoft (tra cui Outlook, Windows e Bot Framework) si sono resi conto che volevano tutti qualcosa di molto simile ("schede") e che ognuno stava progettando soluzioni personalizzate in modo indipendente:</span><span class="sxs-lookup"><span data-stu-id="f7e51-105">In early 2016, several teams at Microsoft (including Outlook, Windows and the Bot Framework) came to the realization that they all wanted something extremely similar ("cards") and that each of them were designing their own solutions independently:</span></span>

- <span data-ttu-id="f7e51-106">Windows aveva il proprio formato dei riquadri animati e delle notifiche</span><span class="sxs-lookup"><span data-stu-id="f7e51-106">Windows had its own Live Tiles and Notifications format</span></span>
-  <span data-ttu-id="f7e51-107">Bot Framework stava usando un set di modelli di scheda predefiniti che gli sviluppatori potevano scegliere per inviare messaggi bot</span><span class="sxs-lookup"><span data-stu-id="f7e51-107">The Bot framework was using a set of predefined card templates developers could choose from when sending Bot messages</span></span>
- <span data-ttu-id="f7e51-108">Outlook stava usando il proprio formato MessageCard per la funzionalità dei messaggi interattivi</span><span class="sxs-lookup"><span data-stu-id="f7e51-108">Outlook was using its own MessageCard format for its Actionable Messages feature</span></span>

<span data-ttu-id="f7e51-109">Allo stesso tempo, altre piattaforme come LINE, FaceBook Messenger, Slack e così via stavano definendo il proprio formato di "scheda" proprietario.</span><span class="sxs-lookup"><span data-stu-id="f7e51-109">At the same time, other platforms such as LINE, FaceBook Messenger, Slack and more were defining their own, proprietary "card" format.</span></span> <span data-ttu-id="f7e51-110">Alcuni dipendenti Microsoft si sono quindi riuniti e hanno iniziato a cercare una soluzione per definire un singolo formato di scheda aperto e un set di SDK in grado di soddisfare i requisiti seguenti:</span><span class="sxs-lookup"><span data-stu-id="f7e51-110">So a few Microsoft employees gathered up and started an effort to define a single, open card format and a set of SDKs that:</span></span>

- <span data-ttu-id="f7e51-111">Facilitare l'interscambio di schede tra host</span><span class="sxs-lookup"><span data-stu-id="f7e51-111">Would facilitate the interchange of cards between hosts</span></span>
- <span data-ttu-id="f7e51-112">Consentire a ogni host di mantenere il controllo dello stile delle schede per garantire la coerenza visiva</span><span class="sxs-lookup"><span data-stu-id="f7e51-112">Would allow each host to keep control over the styling of cards to ensure visual consistency</span></span>
- <span data-ttu-id="f7e51-113">Consentire a un'applicazione host di visualizzare facilmente le schede tramite SDK pronti per l'uso</span><span class="sxs-lookup"><span data-stu-id="f7e51-113">Would make it easy for a host application to display cards with minimum effort via ready-to-use SDKs</span></span>
- <span data-ttu-id="f7e51-114">Fornire infine valore a terze parti e poter essere adottati su larga scala nell'ambito del settore</span><span class="sxs-lookup"><span data-stu-id="f7e51-114">And that would also provide value to third parties and eventually get adopted widely by the industry</span></span>

## <a name="principles-governing-adaptive-cards"></a><span data-ttu-id="f7e51-115">Principi alla base delle schede adattive</span><span class="sxs-lookup"><span data-stu-id="f7e51-115">Principles governing Adaptive Cards</span></span>

1.  <span data-ttu-id="f7e51-116">**Scheda adattiva è un formato di scheda _semplice_ e _dichiarativo_**</span><span class="sxs-lookup"><span data-stu-id="f7e51-116">**Adaptive Card is a _simple_ and _declarative_ card format**</span></span>

    1.  <span data-ttu-id="f7e51-117">Non è concepito come sostituzione/alternativa del formato HTML o XAML</span><span class="sxs-lookup"><span data-stu-id="f7e51-117">It is not meant as an HTML or XAML replacement/alternative</span></span>
    2.  <span data-ttu-id="f7e51-118">Non esiste **alcun "code-behind** " con le schede adattive</span><span class="sxs-lookup"><span data-stu-id="f7e51-118">There is **no "code behind"** with Adaptive Cards</span></span>
        1. <span data-ttu-id="f7e51-119">Gli autori delle schede non possono incorporare codice personalizzato/arbitrario con i payload e, di conseguenza, un host di schede adattive non ha mai la necessità di eseguire codice di terze parti</span><span class="sxs-lookup"><span data-stu-id="f7e51-119">Card authors cannot embed custom/arbitrary code with their payloads, and as a result an Adaptive Card host never needs to run third party code</span></span>

    3.  <span data-ttu-id="f7e51-121">In questo modo si evita un lavoro eccessivo per creare un SDK per schede adattive per una nuova piattaforma</span><span class="sxs-lookup"><span data-stu-id="f7e51-121">This ensures that the effort necessary to build an Adaptive Card SDK for a new platform remains reasonable</span></span>

2.  <span data-ttu-id="f7e51-122">**Il formato Scheda adattiva non può imporre l'uso di una particolare tecnologia sottostante**</span><span class="sxs-lookup"><span data-stu-id="f7e51-122">**The Adaptive Card format can't impose the use of any particular underlying technology**</span></span>

    1.  <span data-ttu-id="f7e51-123">Il formato Scheda adattiva non si basa su JavaScript, C#, Python o un altro linguaggio</span><span class="sxs-lookup"><span data-stu-id="f7e51-123">The Adaptive Card format does not rely on JavaScript, C#, Python or any other language</span></span>
    2.  <span data-ttu-id="f7e51-124">Analogamente, non si basa su HTML o XAML o altri framework grafici o di interfaccia utente</span><span class="sxs-lookup"><span data-stu-id="f7e51-124">Similarly it doesn't rely on HTML or XAML or any other graphic/UI framework</span></span>
    3.  <span data-ttu-id="f7e51-125">È quindi possibile eseguire il rendering delle schede adattive in modo nativo su qualsiasi piattaforma purché esista un renderer</span><span class="sxs-lookup"><span data-stu-id="f7e51-125">This way, Adaptive Cards can be rendered natively on any platform for as long as a renderer exists</span></span>

3.  <span data-ttu-id="f7e51-126">**Il formato Scheda adattiva è una _proprietà condivisa_**</span><span class="sxs-lookup"><span data-stu-id="f7e51-126">**The Adaptive Card format is a _shared property_**</span></span>

    1.  <span data-ttu-id="f7e51-127">Insieme agli SDK corrispondenti, il formato deve essere open source e la sua evoluzione deve essere basata sulla community</span><span class="sxs-lookup"><span data-stu-id="f7e51-127">Along with its SDKs, the format is to be open source and its evolution community driven</span></span>
    2.  <span data-ttu-id="f7e51-128">Il formato non è quindi né di proprietà né gestito da un team specifico</span><span class="sxs-lookup"><span data-stu-id="f7e51-128">The format is therefore not owned nor driven by any one team</span></span>

4.  <span data-ttu-id="f7e51-129">**Il formato Scheda adattiva non è progettato "solo per l'uso da parte di Microsoft"**</span><span class="sxs-lookup"><span data-stu-id="f7e51-129">**The Adaptive Card format is not designed "just for Microsoft's use"**</span></span>

    1.  <span data-ttu-id="f7e51-130">È progettato invece per soddisfare le esigenze di un'ampia gamma di applicazioni e casi d'uso</span><span class="sxs-lookup"><span data-stu-id="f7e51-130">Instead it is designed to suit the needs of a wide variety of applications and use cases</span></span>

    1.  <span data-ttu-id="f7e51-132">Questo gruppo di lavoro è costituito da un gruppo di dipendenti Microsoft profondamente interessati al successo del formato</span><span class="sxs-lookup"><span data-stu-id="f7e51-132">This working group is comprised of a set of Microsoft employees that are all deeply involved in the success of the format</span></span>
    2.  <span data-ttu-id="f7e51-133">Questo gruppo effettua riunioni settimanali (attualmente il lunedì) durante le quali vengono esaminate le proposte di funzionalità, vengono discussi e classificati i problemi aperti e viene tenuta traccia dello stato di avanzamento generale degli elementi di lavoro vNext</span><span class="sxs-lookup"><span data-stu-id="f7e51-133">The Working Group conducts weekly meetings (currently on Monday) during which feature proposals are reviewed, open issues are discussed and triaged and overall progress on vNext work items is tracked</span></span>
    3.  <span data-ttu-id="f7e51-134">Il gruppo di lavoro usa il feedback dell'intera community, inclusi i team Microsoft interni, per decidere in che modo si evolve il formato</span><span class="sxs-lookup"><span data-stu-id="f7e51-134">The Working Group uses feedback from the community at large, including internal Microsoft teams, to decide how the format evolves</span></span>

    4.  <span data-ttu-id="f7e51-137">Per essere approvate dal gruppo di lavoro, le nuove funzionalità proposte devono avere le caratteristiche seguenti:</span><span class="sxs-lookup"><span data-stu-id="f7e51-137">To be approved by the Working Group, proposed new features:</span></span>
        1. <span data-ttu-id="f7e51-138">Essere giustificate da casi d'uso reali</span><span class="sxs-lookup"><span data-stu-id="f7e51-138">Must be justified by real life use cases</span></span>
        2. <span data-ttu-id="f7e51-139">Avere una specifica funzionale</span><span class="sxs-lookup"><span data-stu-id="f7e51-139">Must have a functional specification</span></span>
    5.  <span data-ttu-id="f7e51-140">Le nuove funzionalità approvate vengono aggiunte al backlog e prese in considerazione per vNext</span><span class="sxs-lookup"><span data-stu-id="f7e51-140">Approved new feature are added to the backlog and considered for vNext</span></span>
        1. <span data-ttu-id="f7e51-141">I criteri usati per definire la priorità delle nuove funzionalità includono la gamma di scenari abilitati dalla funzionalità, la complessità, la gestibilità e altro ancora</span><span class="sxs-lookup"><span data-stu-id="f7e51-141">The criteria used to prioritize new features include the breadth of scenarios the feature enables, its complexity/maintainability and more</span></span>

    8.  <span data-ttu-id="f7e51-146">Le nuove versioni del formato e gli SDK passano attraverso una fase beta</span><span class="sxs-lookup"><span data-stu-id="f7e51-146">New versions of the format and SDKs go through a beta phase</span></span>
    9.  <span data-ttu-id="f7e51-147">La pianificazione del rilascio per lo schema delle schede adattive e le versioni degli SDK si basa sulla qualità e non sulla data</span><span class="sxs-lookup"><span data-stu-id="f7e51-147">The release schedule for Adaptive Card schema and SDK versions is driven by quality, not date</span></span>

6.  <span data-ttu-id="f7e51-148">**Interoperabilità**</span><span class="sxs-lookup"><span data-stu-id="f7e51-148">**Interoperability**</span></span>
    1.  <span data-ttu-id="f7e51-149">Le schede create in conformità al formato documentato, ad esempio non usando alcuna estensione specifica dell'host, verranno sottoposte correttamente a rendering in qualsiasi host compatibile con le schede adattive</span><span class="sxs-lookup"><span data-stu-id="f7e51-149">Cards authored in accordance with the documented format (e.g. not using any host-specific extensions) will render properly in any Adaptive Card-capable host</span></span>
